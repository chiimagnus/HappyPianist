# 曲库练习 Ornament 与诊断验收

本清单用于 visionOS Simulator 与 Apple Vision Pro。未实际执行的条目不得标记为通过。

## 曲库布局

- [ ] 主曲库窗口仍保持唱片浏览居中，没有被内部侧边栏压缩。
- [ ] 底部 Ornament 已消失；播放/暂停或导入音频按钮位于曲名与试听进度区域。
- [ ] 右侧 trailing Ornament 始终保持稳定尺寸，只显示练习信息与设置。
- [ ] 曲名、作曲家、来源和试听信息没有在右侧重复。

## 自动准备与状态

- [ ] 切换唱片后立即清除上一首的真实数据并显示系统骨架占位。
- [ ] 快速切换 A → B 时，A 的迟到结果不会覆盖 B。
- [ ] 切回 A 时，未开始的草稿设置被丢弃，并从已保存进度或默认值重建。
- [ ] 无历史进度使用整首、双手、100%、不循环。
- [ ] 有精确 UUID + revision 的历史进度时恢复配置与位置，并保持暂停、静默。

## 设置与进入练习

- [ ] 小节范围、手别、速度、循环和连续成功目标使用同一个 pending configuration。
- [ ] 主按钮始终显示“去练习！”。
- [ ] 配置未改变时进入练习会保留恢复位置。
- [ ] 配置改变后进入练习会从新片段起点开始。
- [ ] 不出现“如何开始练习” dialog、片段 setup sheet 或“从头练习”入口。

## 失败与诊断

- [ ] 损坏 XML、缺失文件、无可练习音符等场景显示具体标题、解释和错误代码。
- [ ] 技术详情默认展开，包含阶段、文件名、App 内相对路径及可用的行列/小节。
- [ ] 技术详情可使用系统文本选择菜单复制。
- [ ] UI 与导出日志中的 code、stage、reason 和 file reference 完全一致。
- [ ] 只有导出存储接受事件后才显示“此错误已写入诊断日志”。
- [ ] “重新准备”生成新的失败事件，不重复使用旧事件 ID。
- [ ] 取消旧选曲任务不显示错误。

## 日志管理与隐私

- [ ] 曲库顶部“诊断”可查看覆盖时间、日志大小、导出与清除操作。
- [ ] 导出 ZIP 只包含 `diagnostics.jsonl`、`diagnostics.txt`、`environment.txt`。
- [ ] 日志保留7 个日历日。
- [ ] 日志不包含绝对路径、原始 MusicXML、逐音 MIDI、音频样本、手部帧、AI 正文或凭据。
- [ ] 导出只由用户触发，不自动上传。

## 无障碍与舒适度

- [ ] Dynamic Type 下内容可滚动，底部“去练习！”保持可达。
- [ ] VoiceOver 能读出骨架加载状态、小节状态、设置和错误技术详情。
- [ ] Reduce Motion 下没有自定义 shimmer 或持续移动动画。
- [ ] Differentiate Without Color 下小节状态仍可通过图标和文字区分。
