# Coaching 教学有效性协议

状态：`pending evidence`。本协议检验一项指导动作是否帮助演奏改善；`issued`、`accepted`、`skipped`、播放次数或点击次数只说明交互发生，不能作为教学有效性结论。

## 单位、冻结与分配

研究单位是一次匿名 participant 的具体 score range 与问题维度。每个单位在开始前冻结 app/score/plan/rubric version、输入 capability、设备/OS、目标范围、before assessment、候选 action、completion condition 与练习剂量定义。

参与者被分配到预先登记的 coaching 条件和对照条件。对照至少保持相同曲目范围、可用时间、回放与 assessment；它可以是不提供该动作的常规练习或已定义的替代动作。分配方式、顺序、交叉条件的 washout 和例外处理必须在收样前记录，不能根据结果挑选对照。

## 每个研究回合

1. **before assessment**：用同一 score range 与 rubric 记录目标维度、evidence status、sample count、confidence 与基线表现；`unknown` / `insufficient` 不当作错误。
2. **action**：仅展示一个有范围和 completion condition 的 coaching action，记录 action kind、tempo ratio、repeat count、reference use、hand/voice focus 与指法来源。
3. **练习剂量**：记录实际完成的重复次数、练习时长桶、session 间隔和中断；没有完成或缺少可比记录时标记 `insufficient`。
4. **after assessment**：在同一范围、相同或明确可比条件下复测；系统内的 remeasure 只验证 completion condition，研究报告仍须保留 before/after 的完整分维度结果。
5. **迁移与副作用**：另测完整段落或未直接练习的相邻段落，并记录目标恶化、其他维度退步、错误迁移、练习负担或安全/可访问性问题。

## 结果与边界

按 pitch、timing、duration、dynamics、voicing、pedal 与对应 action kind 分层报告：目标问题的改善、完成条件达成、完整段落迁移、对照差异、有效/unknown/insufficient 样本数和副作用。分析方法、阈值和最小样本量在收样前绑定到 protocol/rubric version；不得只报告改善的子集或用总体平均掩盖退步。

应用的低频诊断可帮助追踪 action 是否被发出、接受、跳过和复测，但其设计不保存逐音演奏或参与者身份，也不构成研究证据。研究记录只使用匿名 ID 与聚合结果，并另行取得参与者授权。

记录模板：

```text
状态：pending / passed / blocked
protocol / app / score / plan / rubric version：
匿名 participant ID、设备、OS、input capability：
score range、目标维度与 before assessment：
coaching / control condition、action 与 completion condition：
练习剂量、缺失与 insufficient：
after assessment、完整段落 transfer、负面副作用：
分层结果、阈值、对照差异与证据位置：
```
